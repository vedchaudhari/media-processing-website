# 2. API Client & Shared Types

**Status:** ✅ Done

### Types (`src/lib/types.ts`)

- `VideoStatus` — mirrors backend's 9-state lifecycle
- `VideoListItem` — shape from `GET /get-videos`
- `InitiateUploadResponse`, `CompleteUploadResponse` — upload flow responses
- `PlayResult` — normalized play response (`ready: true` = playable, `ready: false` = still processing)
- `isInProgress()` helper — returns true for any active pipeline status (used for polling/spinners)

### API Client (`src/lib/api.ts`)

- `request<T>()` — generic fetch wrapper; throws with status + backend error message on non-2xx
- `initiateUpload(title)` — POST to get presigned URL
- `uploadToStorage(url, file, onProgress)` — **XHR-based** direct PUT to MinIO (XHR used because `fetch` doesn't support upload progress events); calls `onProgress(0–100)`
- `completeUpload(videoId)` — POST to confirm upload
- `listVideos()` — GET all videos
- `getPlay(videoId)` — GET play info; treats `409` as `{ ready: false }` instead of an error (so the UI can show processing/failed state without error boundaries)
