# 10. Interactive Transcript UI

**Status:** ✅ Done

- **Synced transcript panel:** Renders below the player on the video detail page once the video is ready, driven by the `transcript` object returned from `GET /:id/play`.
- **Status-aware rendering:** Distinct UI for each transcript state:
  - `pending` → "Waiting for transcription to start…"
  - `processing` → spinner + "Transcribing audio in the background…"
  - `failed` → red error card showing the saved error message
  - `completed` → scrollable list of timestamped segments
  - no transcript → "No transcript available."
- **Live highlighting:** Subscribes to the Video.js `timeupdate` event (via `onReady` → `playerRef`); the segment whose `[start, end)` range contains the current playback time is highlighted in real time.
- **Click-to-seek:** Clicking any segment seeks the player to that segment's `start` and resumes playback.
- **Polling integration:** The detail page keeps polling every 3s while the video is ready but the transcript is still `pending`/`processing`, so the panel fills in automatically without a manual refresh (see entry 5).
- Timestamps formatted as `m:ss`; segment list capped to a scrollable max height.

### Key Files

- `src/app/videos/[id]/VideoDetail.tsx` — transcript panel, highlight + seek logic
- `src/lib/types.ts` — `ITranscript` / `ITranscriptSegment`, `PlayResult.transcript`
- `src/lib/api.ts` — `getPlay` returns `transcript` for both ready and processing states
