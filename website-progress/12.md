# 12. AI Chapters UI

**Status:** ✅ Done

- **Chapters list on the detail page:** Renders at the top of the **AI Insights** tab (inside the `aiSummary.status === "completed"` block), above the Summary section.
- **Click-to-seek:** Each chapter is a button that calls the existing `seekTo(start)` to jump the Video.js player to that timestamp; timestamps use the existing `formatTime` helper.
- **Live highlighting:** The chapter currently playing is highlighted by comparing `currentTime` against `[chapter.start, nextChapter.start)`, mirroring the interactive-transcript behaviour.
- **Graceful absence:** Guarded by `chapters?.length > 0`, so videos processed before the feature (or ones where the model returned none) simply show no chapters section.
- **No new fetch/polling:** Chapters arrive inside the same `aiSummary` object the page already fetches and polls for, so `api.ts` and the poll logic were untouched.
- **Type mirror:** Added `IChapter` and `chapters` to `IAISummary` in the shared types to match the backend contract (see backend entry 16).
